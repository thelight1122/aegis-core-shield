# ACTIVATION_PROTOCOL.md

AEGIS Protocol — Activation & Automation Protocol (v1.0)

## 0. Fundamental Posture

The AEGIS system operates on a **non-force governance** basis. Activation of any autonomous or semi-autonomous behavior must never transition into enforcement, coercion, or punishment. Activation exists purely to support human awareness and preserve system continuity.

## 1. Definitions

### 1.1 NCT (Nostalgic Context Tensor)

The NCT serves as the system's "memory-of-meaning." It stores high-level summaries, experiential groupings, and historical context. It is the narrative substrate that justifies pattern recognition.

### 1.2 SPINE (Stabilized Patterned Interpretive Nexus)

The SPINE represents the stabilized "skeleton" of the system's understanding. It stores invariants, structural relationships, and slow-earned patterns. An entry in the SPINE indicates that a pattern has reached a threshold of stability.

## 2. Protocol for Activation

### 2.1 Consent & Sovereignty

- No activation logic may be executed without explicit, auditable consent recorded in the **PCT (Persistent Context Tensor)**.
- Humans retain absolute sovereignty; any system-suggested posture or automation can be toggled OFF or reversed instantly.

### 2.2 Threshold of Evidence

- Before a pattern is "activated" (used for summarization or predictive scaffolding), it must exist in the **SPINE** with an `invariant: true` flag or substantial evidence of stability.
- Activation must be accompanied by an **NCT** record detailing the historical context that warrants the state change.

### 2.3 Visibility Requirements

- Every activated action must log a transition to the **Boundary Ledger (`boundary.jsonl`)** via the Runtime Witness.
- The system must provide a human-legible "Why" (referencing NCT/SPINE) for every automated suggestion.

### 2.4 Reversibility

- Any state change resulting from an activation must be compensatable via a new append-only ledger entry.
- Overwriting historical state is strictly forbidden.

## 3. Forbidden Behaviors

- **Enforcement**: No activation may block, restrict, or penalty human behavior.
- **Hidden State**: Activation logic must be visible in the source code; no "black-box" decision weights are permitted.
- **Automatic Truth**: The system must never claim its interpretations are "objective truth"; they remain "observed patterns" with inherent uncertainty.

## 4. Activation Tensors

- **NCT**: Provides the "Why" (Context).
- **SPINE**: Provides the "What" (Stable Pattern).
- **PCT**: Provides the "Who/When" (Consent and Active Parameters).
- **PEER**: provides the raw "Experience" (Evidence).
